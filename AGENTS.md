# 开发原则

后续所有改动前，先读取本文件，再开始修改。

## 必须遵守

1. 先保证微信开发者工具可以编译并进入页面，再考虑功能扩展。
2. 不擅自改动现有业务算法、常量口径、计算服务，除非用户明确要求。
3. 修改小程序页面时，先检查该页面实际入口文件是否同时存在 `.js` 和 `.ts`。
4. 如果 `.js` 和 `.ts` 同时存在，不能只改一个；必须保持两者逻辑一致，或在确认安全后移除歧义入口。
5. 改动前先确认 `app.json` 页面注册路径、页面目录结构、脚本文件名是否匹配。
6. 优先做最小修复，避免为了小问题扩散到无关模块。
7. 每次修复完成后，至少检查：
   - 页面路径是否存在
   - 注册路径是否正确
   - 入口脚本是否存在
   - 没有因 `.js` / `.ts` 不匹配引入新报错

## 当前项目特别注意

- 这是微信原生小程序项目，源码根目录是 `miniprogram/`。
- 当前工程里部分页面曾出现过编码异常和 `.js` / `.ts` 并存问题。
- 对运行链路上的页面，默认先检查：`index.json`、`index.wxml`、`index.wxss` / `index.less`、`index.js`、`index.ts`。
- 运行链路上的页面修改后，优先在微信开发者工具验证首页跳转、页面渲染和按钮事件。
